🔬 Quantization Technology Explained

What is Quantization?

Quantization is the technique of converting model weights from high precision (e.g., FP16, 16-bit floating point) to low precision (e.g., INT4, 4-bit integer). The core formula:

Quantized Weight = round(Original Weight × Scale Factor) / Scale Factor

4-bit vs 8-bit vs FP16

Quantization Level VRAM Usage Accuracy Loss Inference Speed Use Cases
FP16 100% None Baseline Production, high precision needs
8-bit 50% <1% +20% Balanced performance and cost
4-bit 25% 1-3% +50% Budget constraints, rapid prototyping
Real-world Data: In code generation tasks, 4-bit quantized Llama 3 8B shows only 2.1% accuracy drop compared to FP16, while VRAM usage drops from 16GB to 6GB.

⚠️ Pitfall Guide: Common Quantization Misconceptions

Misconception 1: Quantization Severely Degrades Model Quality

Fact: Modern quantization techniques (like GGUF, AWQ) use group-wise quantization and per-channel scaling. Quality loss for 4-bit is typically under 3%. For customer service, document summarization, and similar tasks, users can barely notice the difference.

Misconception 2: Quantization Slows Down Inference

Fact: Quite the opposite! 4-bit models have smaller data footprints and lower memory bandwidth requirements, achieving 30-50% faster inference on the same hardware. The bottleneck is dequantization computation, but modern GPU Tensor Cores are optimized for low precision.

Misconception 3: All Models are Suitable for 4-bit

Caution: Use 8-bit or FP16 for these scenarios:

  • Mathematical computation and logical reasoning tasks
  • High-risk domains like healthcare and legal
  • Scenarios requiring precise numerical output
  • Very small models (<3B parameters)

📊 VRAM Optimization Tips

1. KV Cache Optimization

Enabling Grouped Query Attention (GQA) can reduce KV Cache usage by 50%:

# Ollama Configuration
OLLAMA_NUM_CTX=4096  # Context length
OLLAMA_MAX_VRAM=8GB  # VRAM limit

2. Model Parallel Loading

When single GPU VRAM is insufficient, split the model across multiple GPUs:

# Using accelerate library
from accelerate import init_empty_weights, load_checkpoint_and_dispatch

with init_empty_weights():
    model = AutoModelForCausalLM.from_pretrained("model_name")

model = load_checkpoint_and_dispatch(
    model,
    checkpoint="model_weights.bin",
    device_map="auto",  # Auto-distribute across GPUs
    max_memory={0: "12GB", 1: "12GB"}  # Per-GPU VRAM limit
)

3. Offload to CPU Memory

Use llama.cpp to offload some layers to CPU, trading speed for VRAM:

# Offload 20 layers to CPU
./main -m model.gguf -ngl 20 --n-gpu-layers 20
