1. 内存管理优化 (Memory Management)LLM 权重通常达到数十 GB，内核的内存分配机制直接影响加载速度和推理稳定性。启用透明大页 (Transparent Huge Pages, THP):大页可以显著减少 TLB（Translation Lookaside Buffer）未命中次数，从而提升大内存访问效率。建议： 将 /sys/kernel/mm/transparent_hugepage/enabled 设置为 madvise 或 always。调整 Swap 置换倾向 (Swappiness):推理过程中如果触发 Swap 会导致严重的卡顿。建议： 将 vm.swappiness 设置为 1 或 10，迫使内核尽量留在物理内存中。内存锁定 (MLOCK):在应用层使用 mlockall()，并确保内核限制（ulimit）足够大，防止模型权重被交换到磁盘。2. 存储与 I/O 优化 (Storage & I/O)加载 70B 规模的模型需要从磁盘读取大量数据，I/O 吞吐量是瓶颈。使用性能模式的 I/O 调度器:对于 NVMe 固态硬盘，通常建议使用 none 或 mq-deadline。预读 (Read-ahead) 调整:在加载大文件时，增加块设备的预读参数可以提升顺序读取速度。blockdev --setra 4096 /dev/nvme0n1减少文件系统开销:使用 noatime 挂载选项，减少读取时的元数据更新开销。3. 处理器调度与电源管理 (CPU & Power)即便使用 GPU 推理，CPU 的调度延迟也会影响算子分发和 KV Cache 的管理。CPU Governor 设置:将 CPU 频率调节器设置为 performance 模式，避免内核在推理间隙动态调频导致延迟。cpupower frequency-set -g performance内核抢占模式:如果追求极低的推理延迟，可以使用 PREEMPT_LL（低延迟抢占）或 PREEMPT_RT 内核，但对于通用推理，标准内核通常足够。4. 关键内核参数调整 (Sysctl)通过修改 /etc/sysctl.conf 来优化并行处理能力：参数建议值说明kernel.sched_min_granularity_ns10000000减少 CPU 线程频繁切换vm.max_map_count655300某些推理框架（如 llama.cpp）需要大量内存映射fs.file-max2097152增加最大文件句柄数5. GPU 专用内核环境 (GPU & Direct I/O)启用 GPUDirect Storage (GDS):如果你的硬件支持（如 NVIDIA A100/H100），确保内核支持并配置了 NVIDIA 的 nvidia-fs 驱动。这允许数据绕过 CPU 内存，直接从 NVMe 加载到 GPU 显存。PCIe 最大载荷 (Max Payload):在 BIOS 和内核引导参数中确保 PCIe 的传输效率最大化。6. 监控工具在优化过程中，可以使用以下内核级监控工具定位瓶颈：perf: 检查内核函数耗时。iotop: 监控磁盘 I/O 峰值。htop / btop: 实时观察核调度。总结优化的黄金组合通常是：开启 Huge Pages + Performance Governor + 禁用 Swapping + 优化 NVMe 预读。
