🤖 AI Prompt Engineering: 如何写出完美的 Shell 脚本生成指令

阅读时长：约 18 分钟
掌握提示词工程，让 AI 帮你写出高质量、安全的 Bash 脚本

什么是 Prompt Engineering？

Prompt Engineering（提示词工程）是指通过精心设计的输入提示，引导 AI 模型生成更准确、更有用的输出结果。

Shell 脚本生成的核心要素

1. 角色设定

明确 AI 的角色，让它以专业视角思考：

作为拥有 10 年经验的 Linux 系统管理员...
作为 DevOps 专家，擅长自动化脚本...
作为安全工程师，专注于脚本安全性...

2. 任务描述

要素示例
目标备份 /var/www 目录
约束不超过 50 行代码
格式使用 Bash 4.0+ 语法
输出生成完整可执行脚本

3. 安全要求

请确保脚本：
- 验证输入参数
- 检查命令执行结果
- 不使用 eval
- 正确处理特殊字符
- 包含错误处理

完整 Prompt 模板

模板一：系统管理脚本

作为高级 Linux 运维专家，请编写一个 [任务描述] 脚本。

具体要求：
1. 使用 Bash 4.0+ 语法
2. 包含参数验证
3. 实现错误处理（set -euo pipefail）
4. 添加详细注释说明每个步骤
5. 遵循 ShellCheck 最佳实践
6. 包含使用示例

请输出完整的、可直接运行的脚本代码。

实战示例：数据库备份脚本

#!/bin/bash
#===============================================================================
# 脚本名称：mysql_backup.sh
# 功能描述：MySQL 数据库生产环境备份脚本
#===============================================================================

set -euo pipefail  # 严格错误处理

BACKUP_DIR="${1:-${MYSQL_BACKUP_DIR:-/backup/mysql}}"
RETENTION_DAYS="${MYSQL_RETENTION_DAYS:-7}"

log() {
    echo "[$(date '+%Y-%m-%d %H:%M:%S')] $*" | tee -a "$LOG_FILE"
}

error_exit() {
    log "ERROR: $*"
    exit 1
}

check_prerequisites() {
    if [[ -z "${MYSQL_PWD:-}" ]]; then
        error_exit "MYSQL_PWD 环境变量未设置"
    fi
    if ! command -v mysqldump &>/dev/null; then
        error_exit "mysqldump 命令未找到"
    fi
}

perform_backup() {
    local timestamp=$(date '+%Y%m%d_%H%M%S')
    local backup_file="${BACKUP_DIR}/backup_${timestamp}.sql.gz"
    log "开始备份数据库..."
    mysqldump -u "$MYSQL_USER" --all-databases | gzip > "$backup_file"
    log "备份完成：$backup_file"
}

main() {
    log "MySQL 备份任务开始"
    check_prerequisites
    perform_backup
    log "备份任务成功完成"
}

main "$@"

进阶技巧

1. 少样本学习（Few-Shot Learning）

给 AI 提供示例，让它理解你的期望。

2. 链式思考（Chain of Thought）

让 AI 分步骤思考：1) 分析需求 2) 设计结构 3) 编写代码 4) 添加错误处理 5) 补充注释

结语

掌握 Prompt Engineering 可以大幅提升你的 Shell 脚本开发效率。记住：具体胜于模糊，安全优先，迭代优化，人工审核。
