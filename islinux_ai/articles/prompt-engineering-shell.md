# AI Prompt Engineering: 如何写出完美的 Shell 脚本生成指令

> 掌握提示词工程，让 AI 帮你写出高质量、安全的 Bash 脚本

## 什么是 Prompt Engineering？

Prompt Engineering（提示词工程）是指通过精心设计的输入提示，引导 AI 模型生成更准确、更有用的输出结果。

### 为什么重要？

```
❌ 糟糕的提示词：
"写一个备份脚本"

✅ 优秀的提示词：
"作为高级 Linux 运维工程师，请编写一个生产环境的 MySQL 数据库备份脚本。
要求：1) 使用 mysqldump 2) 自动压缩 3) 保留 7 天备份 4) 错误日志记录 5) 包含详细注释"
```

## Shell 脚本生成的核心要素

### 1. 角色设定

明确 AI 的角色，让它以专业视角思考：

```
作为拥有 10 年经验的 Linux 系统管理员...
作为 DevOps 专家，擅长自动化脚本...
作为安全工程师，专注于脚本安全性...
```

### 2. 任务描述

清晰、具体地描述需求：

| 要素 | 示例 |
|------|------|
| 目标 | "备份 /var/www 目录" |
| 约束 | "不超过 50 行代码" |
| 格式 | "使用 Bash 4.0+ 语法" |
| 输出 | "生成完整可执行脚本" |

### 3. 安全要求

Shell 脚本涉及系统操作，必须强调安全：

```
请确保脚本：
- 验证输入参数
- 检查命令执行结果
- 不使用 eval
- 正确处理特殊字符
- 包含错误处理
```

### 4. 注释规范

```
要求：
- 每个函数都有文档字符串
- 关键步骤添加行内注释
- 包含使用示例
```

## 实战示例

### 示例一：数据库备份脚本

**Prompt**：
```
作为高级 Linux 运维工程师，请编写一个生产环境的 MySQL 数据库备份脚本。

环境信息：
- MySQL 8.0
- 备份存储：/backup/mysql
- 保留策略：7 天
- 压缩方式：gzip

功能要求：
1. 支持全量备份
2. 自动删除过期备份
3. 记录备份日志
4. 失败时发送告警（邮件）
5. 支持自定义备份目录

安全要求：
- 从环境变量读取密码
- 验证磁盘空间
- 检查 MySQL 服务状态

请生成完整脚本，包含详细注释和使用说明。
```

**AI 生成结果**：
```bash
#!/bin/bash
#===============================================================================
# 脚本名称：mysql_backup.sh
# 功能描述：MySQL 数据库生产环境备份脚本
# 使用方式：./mysql_backup.sh [备份目录]
#===============================================================================

set -euo pipefail  # 严格错误处理

# 配置项（可通过环境变量覆盖）
BACKUP_DIR="${1:-${MYSQL_BACKUP_DIR:-/backup/mysql}}"
RETENTION_DAYS="${MYSQL_RETENTION_DAYS:-7}"
MYSQL_USER="${MYSQL_USER:-backup}"
LOG_FILE="/var/log/mysql_backup.log"

#-------------------------------------------------------------------------------
# 日志函数
#-------------------------------------------------------------------------------
log() {
    echo "[$(date '+%Y-%m-%d %H:%M:%S')] $*" | tee -a "$LOG_FILE"
}

#-------------------------------------------------------------------------------
# 错误处理函数
#-------------------------------------------------------------------------------
error_exit() {
    log "ERROR: $*"
    # 这里可以添加邮件告警逻辑
    exit 1
}

#-------------------------------------------------------------------------------
# 前置检查
#-------------------------------------------------------------------------------
check_prerequisites() {
    # 检查 MySQL 密码是否设置
    if [[ -z "${MYSQL_PWD:-}" ]]; then
        error_exit "MYSQL_PWD 环境变量未设置"
    fi

    # 检查 mysqldump 是否可用
    if ! command -v mysqldump &>/dev/null; then
        error_exit "mysqldump 命令未找到"
    fi

    # 检查磁盘空间（至少需要 1GB）
    local available_space
    available_space=$(df -P "$BACKUP_DIR" | awk 'NR==2 {print $4}')
    if [[ "$available_space" -lt 1048576 ]]; then
        error_exit "磁盘空间不足（需要至少 1GB）"
    fi
}

#-------------------------------------------------------------------------------
# 主备份函数
#-------------------------------------------------------------------------------
perform_backup() {
    local timestamp
    timestamp=$(date '+%Y%m%d_%H%M%S')
    local backup_file="${BACKUP_DIR}/backup_${timestamp}.sql.gz"

    log "开始备份数据库..."

    # 获取所有数据库列表
    local databases
    databases=$(mysql -u "$MYSQL_USER" -e "SHOW DATABASES;" | grep -Ev "(Database|information_schema|performance_schema)")

    for db in $databases; do
        log "备份数据库：$db"
        mysqldump -u "$MYSQL_USER" --databases "$db" | gzip >> "$backup_file"
    done

    log "备份完成：$backup_file"
}

#-------------------------------------------------------------------------------
# 清理过期备份
#-------------------------------------------------------------------------------
cleanup_old_backups() {
    log "清理 ${RETENTION_DAYS} 天前的备份..."
    find "$BACKUP_DIR" -name "backup_*.sql.gz" -mtime +${RETENTION_DAYS} -delete
    log "清理完成"
}

#-------------------------------------------------------------------------------
# 主程序
#-------------------------------------------------------------------------------
main() {
    log "=========================================="
    log "MySQL 备份任务开始"
    log "=========================================="

    check_prerequisites
    perform_backup
    cleanup_old_backups

    log "备份任务成功完成"
}

main "$@"
```

## 进阶技巧

### 1. 少样本学习（Few-Shot Learning）

给 AI 提供示例，让它理解你的期望：

```
参考以下脚本风格：

示例 1：
```bash
# 函数命名使用下划线
# 变量使用大写
# 每个函数前有注释块
```

请按照相同风格编写...
```

### 2. 链式思考（Chain of Thought）

让 AI 分步骤思考：

```
请按以下步骤完成任务：
1. 首先分析需求，列出关键点
2. 设计脚本结构
3. 编写核心函数
4. 添加错误处理
5. 补充注释和文档
```

### 3. 约束条件

明确限制可以避免 AI 生成不切实际的代码：

```
约束条件：
- 只能使用 POSIX 兼容命令
- 不依赖外部工具（除了标准 coreutils）
- 脚本不超过 100 行
- 兼容 Bash 3.2+
```

## 常见错误及避免方法

### 1. 过于模糊

```
❌ "写个脚本处理文件"
✅ "写一个脚本，将 /var/log 下所有 .log 文件压缩并移动到 /archive"
```

### 2. 忽略错误处理

```
❌ 不提及错误处理
✅ 明确要求 "使用 set -euo pipefail，每个命令检查返回值"
```

### 3. 缺少安全考虑

```
❌ 直接生成可执行代码
✅ 要求 "验证所有输入，转义特殊字符，不使用 eval"
```

## 结语

掌握 Prompt Engineering 可以大幅提升你的 Shell 脚本开发效率。记住：

1. **具体胜于模糊** - 详细描述需求
2. **安全优先** - 始终要求错误处理
3. **迭代优化** - 根据结果调整提示词
4. **人工审核** - AI 生成的代码需要审查
