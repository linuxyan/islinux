📁 Essential Linux File and Directory Commands

10 min read

Introduction

File and directory management is at the core of Linux system operations. Whether you're organizing project files, managing system configurations, or automating backups, mastering these commands is essential. This guide covers 10 fundamental commands for file and directory operations.

1. mkdir - Create Directories

Create new directories with various options.

# Create a single directory
mkdir projects

# Create multiple directories
mkdir dir1 dir2 dir3

# Create nested directories
mkdir -p projects/backend/src

# Create directory with permissions
mkdir -m 750 secure_folder

# Verbose output
mkdir -v new_directory

Common Options:

  • -p: Create parent directories
  • -m: Set permissions
  • -v: Verbose output

2. rmdir - Remove Empty Directories

Remove empty directories only.

# Remove empty directory
rmdir empty_folder

# Remove multiple empty directories
rmdir dir1 dir2 dir3

# Remove nested empty directories
rmdir -p parent/child/grandchild

Note: rmdir only works on empty directories. Use rm -r for non-empty directories.

3. cp - Copy Files and Directories

Copy files and directories with various options.

# Copy a file
cp file.txt backup.txt

# Copy to another directory
cp file.txt /home/user/backup/

# Copy directory recursively
cp -r source_dir/ dest_dir/

# Copy with verbose output
cp -v file.txt /backup/

# Copy preserving attributes
cp -p important.txt backup/

# Interactive copy
cp -i file.txt existing.txt

Common Options:

  • -r: Recursive copy
  • -p: Preserve attributes
  • -v: Verbose output
  • -i: Interactive
  • -u: Copy only when newer

4. mv - Move and Rename Files

Move files/directories or rename them.

# Rename a file
mv oldname.txt newname.txt

# Move file to another directory
mv file.txt /home/user/documents/

# Move and rename
mv file.txt /backup/file_backup.txt

# Interactive move
mv -i file.txt /existing/

# Verbose output
mv -v file.txt /backup/

5. rm - Remove Files and Directories

Delete files and directories permanently.

# Remove a file
rm file.txt

# Remove multiple files
rm file1.txt file2.txt

# Remove directory and contents
rm -r directory/

# Force remove
rm -f file.txt

# Interactive remove
rm -ri directory/

# Verbose output
rm -v file.txt

WARNING: rm -rf is dangerous. Always double-check the path!

6. ls - List Directory Contents

Display files and directories with various formats.

# Basic listing
ls

# Detailed listing
ls -l

# Show hidden files
ls -a

# Human-readable sizes
ls -lh

# Sort by time
ls -lt

# Sort by size
ls -lS

# Show inode numbers
ls -li

🚀 Modern Alternatives: exa and lsd (2026 Recommendation)

exa and lsd are modern ls replacements with icons, colors, Git integration, and tree view. Perfect for visual file browsing and AI project navigation.

# Install exa
sudo apt install exa  # Debian/Ubuntu
brew install exa      # macOS

# Install lsd
cargo install lsd     # Rust

# Colorful listing with icons
exa -lah
lsd -lah

# Tree view for project structure
exa --tree -L 3
lsd --tree -d 3

# Git status integration
exa --git
lsd --git

# AI project example: View model directory
exa --tree -L 2 /models/llama-3/
lsd -lh /datasets/

Why use exa/lsd in 2026? Better visual hierarchy with icons, Git status overlay, and tree view make them ideal for navigating complex AI project structures and model directories.

💡 AI Use Case: Managing LLM Project Files

When working with local LLM deployments, organizing model files, datasets, and outputs requires efficient directory management:

# Create AI project structure
mkdir -p llm-project/{models,datasets,outputs,configs}

# View model directory with tree
exa --tree -L 2 llm-project/models/

# Find all GGUF model files
find ~/models -name "*.gguf" -type f

# Find large output files (>100MB)
find ~/llm-outputs -type f -size +100M

# Clean up old checkpoint files
find ./training -name "checkpoint_*" -mtime +7 -delete

# List files sorted by modification time
lsd -lt ~/llm-project/outputs/

# View directory sizes
du -sh */ | sort -h

7. cd - Change Directory

Navigate between directories.

# Change to specific directory
cd /var/log

# Change to home directory
cd

# Change to parent directory
cd ..

# Change to previous directory
cd -

# Change using ~ shortcut
cd ~/documents

8. pwd - Print Working Directory

Display the current directory path.

# Show current directory
pwd

# Show resolved path (no symlinks)
pwd -P

# Show logical path
pwd -L

9. ln - Create Links

Create hard or symbolic links to files.

# Create symbolic link
ln -s /path/to/file linkname

# Create hard link
ln file.txt hardlink.txt

# Verbose output
ln -sv source target

10. find - Search for Files

Search for files with various criteria.

# Find by name
find /home -name "*.txt"

# Find by type (file)
find /var -type f

# Find by type (directory)
find /var -type d

# Find files larger than 100MB
find / -size +100M

# Find files modified in last 7 days
find /home -mtime -7

# Find and delete old logs
find /var/log -name "*.log" -mtime +30 -delete

Conclusion
