📄 Essential Linux File Content Viewing Commands

10 min read

Introduction

Viewing and analyzing file contents is a daily task for Linux users. From checking log files to examining configuration files, these commands help you quickly access and understand file contents. This guide covers 10 essential commands for viewing file contents.

1. cat - Concatenate and Display

Display entire file contents or concatenate multiple files.

# Display file content
cat file.txt

# Display multiple files
cat file1.txt file2.txt

# Show line numbers
cat -n file.txt

# Show non-printable characters
cat -v file.txt

# Create a file (Ctrl+D to save)
cat > newfile.txt

# Append to file
cat >> existing.txt

🚀 Modern Alternative: bat (2026 Recommendation)

bat is a modern cat clone with syntax highlighting, Git integration, and pager support. Perfect for viewing code files and configuration files with better readability.

# Install bat
sudo apt install bat  # Debian/Ubuntu
brew install bat      # macOS

# View file with syntax highlighting
bat config.yaml

# Show line numbers and Git changes
bat -n --show-all /etc/nginx/nginx.conf

# View specific lines
bat -r 10:50 app.py

# Compare with cat mode
bat -A file.txt  # Show non-printable characters

Why use bat in 2026? Syntax highlighting for 50+ languages, Git diff integration, and built-in pager make it ideal for reviewing AI-generated code and configuration files.

💡 AI Use Case: Analyzing LLM Output Logs

When working with local LLM deployments, you often need to analyze large output logs:

# View AI response with syntax highlighting
bat llm_output.json

# Search for specific patterns in logs
grep -A 5 "completion" llm_output.log | bat -l json

# Count tokens in output file
wc -w llm_output.txt

# View last 100 lines of generation
tail -n 100 llm_output.log | bat -l markdown

2. less - View Page by Page

View large files with scrollable pagination.

# View file with pagination
less file.txt

# Open at line 100
less +100 file.txt

# Search for pattern on open
less -p "error" file.txt

# View with line numbers
less -N file.txt

Navigation: Space=next page, b=previous page, g=beginning, G=end, /=search, q=quit

3. head - View First Lines

Display the beginning of a file.

# Show first 10 lines (default)
head file.txt

# Show first 20 lines
head -n 20 file.txt

# Show first 5 lines
head -5 file.txt

# Show multiple files
head file1.txt file2.txt

4. tail - View Last Lines

Display the end of a file, useful for logs.

# Show last 10 lines (default)
tail file.txt

# Show last 50 lines
tail -n 50 file.txt

# Follow file in real-time
tail -f /var/log/syslog

# Follow with retry
tail -F /var/log/application.log

Tip: Press Ctrl+C to exit tail -f.

5. nano - Simple Text Editor

Edit files with a beginner-friendly editor.

# Open file for editing
nano file.txt

# Open at specific line
nano +10 file.txt

# Read-only mode
nano -v file.txt

Shortcuts: Ctrl+O=Save, Ctrl+X=Exit, Ctrl+W=Search, Ctrl+_=Go to line

6. vim - Advanced Text Editor

Powerful editor for advanced users.

# Open file for editing
vim file.txt

# Open at specific line
vim +100 file.txt

# Read-only mode
vim -R file.txt

# Compare two files
vimdiff file1.txt file2.txt

Basic: i=insert, Esc=normal, :w=save, :q=quit, :wq=save and quit

7. grep - Search Text Patterns

Search for patterns in files.

# Search for pattern
grep "error" file.txt

# Case-insensitive search
grep -i "error" file.txt

# Show line numbers
grep -n "error" file.txt

# Invert match
grep -v "debug" file.txt

# Count matches
grep -c "error" file.txt

# Show context (3 lines)
grep -C 3 "error" file.txt

# Recursive search
grep -r "function" /home/user/

# Extended regex
grep -E "error|warning" file.txt

8. wc - Word Count

Count lines, words, and characters.

# Count lines, words, characters
wc file.txt

# Count lines only
wc -l file.txt

# Count words only
wc -w file.txt

# Count characters only
wc -m file.txt

9. diff - Compare Files

Show differences between files.

# Compare two files
diff file1.txt file2.txt

# Unified format
diff -u file1.txt file2.txt

# Compare directories
diff -r dir1/ dir2/

# Ignore whitespace
diff -w file1.txt file2.txt

# Side-by-side comparison
diff -y file1.txt file2.txt

# Create patch file
diff -u original.txt modified.txt > changes.patch

# Apply patch
patch original.txt < changes.patch

10. sort - Sort Lines

Sort lines of text files.

# Basic sort
sort file.txt

# Reverse sort
sort -r file.txt

# Numeric sort
sort -n file.txt

# Remove duplicates
sort -u file.txt

# Sort by field
sort -t: -k2 /etc/passwd
